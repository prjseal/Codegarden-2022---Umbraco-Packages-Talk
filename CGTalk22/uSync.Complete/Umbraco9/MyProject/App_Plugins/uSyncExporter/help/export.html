A sync pack is the collection of configuration and files needed to recreate settings and content on an Umbraco installation

Pack Elements

you can select any number of elements to include in your sync pack

Include media files

Include the media files associated with any media items that are synced.

Include website files

Include all files required to display the site, (templates, css, scripts)

If you tick this box you get all files, regardless of what templates, or scripts you pick from the dashboard. If you only want to include some files, you should use the pickers and leave this box unticked.

Include node config

Included the configuration associated with any content node, e.g Culture and Hostnames, or Public access settings.

Include linked items

Include all items that are linked to the items you have selected, e.g if you pick a single page all the pages linked to on that page via the content or content pickers, will be included in the export.

Including items that are linked from other items, can result in large sync files, especially if it is also combined with other settings such as include children / ancestors as all items linked in those elsements will also be calculated.

Dependencies

Elements that are needed for any selected item, (e.g datatypes for a doctype)

Including dependencies means the whole item can be recreated on another umbraco instance, you should only uncheck this option if you are sure the target umbraco installation has all the configuration needed to recreate an item.

Children

Any elements that live below this element within the tree

Ancestors

The parent elments for an item, including these will build the tree when you import into a new instance
